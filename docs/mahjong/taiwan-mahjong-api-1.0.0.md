# 台式麻將胡牌判斷 API 1.0.0 規格書

本文件定義了用於判斷台式麻將（16 張）是否胡牌的 API 規格。

## 1. 概述

台式麻將通常為 16 張牌，胡牌時含最後補進的一張牌共 17 張。此 API 接收一組手牌，並根據台式麻將的胡牌規則（五面子 + 一對子）判斷是否胡牌。

## 2. API 端點

### 判斷是否胡牌

*   **URL:** `/api/mahjong/check-winning`
*   **Method:** `POST`
*   **Content-Type:** `application/json`

## 3. 請求參數

### JSON Body

| 參數名    | 類型              | 必填 | 說明           |
|:-------|:----------------|:---|:-------------|
| `hand` | `array<string>` | 是  | 包含 17 張牌的陣列。 |

### 牌的表示法 (Tile Notation)

使用字串表示每一張牌：
*   **萬子 (Wan):** `1w`, `2w`, ..., `9w`
*   **筒子 (Tong):** `1t`, `2t`, ..., `9t`
*   **條子 (Tiao):** `1s`, `2s`, ..., `9s`
*   **字牌 (Zhi):**
    *   東南西北: `east`, `south`, `west`, `north`
    *   中發白: `red`, `green`, `white`

#### 範例請求
```json
{
  "hand": [
    "1w", "2w", "3w",
    "4w", "5w", "6w",
    "7t", "8t", "9t",
    "2s", "2s", "2s",
    "5s", "5s", "5s",
    "east", "east"
  ]
}
```

## 4. 回應格式

### 成功回應 (胡牌)

*   **Status Code:** `200 OK`

```json
{
  "is_winning": true,
  "message": "恭喜胡牌！",
  "combination": {
    "mianzi": [
      ["1w", "2w", "3w"],
      ["4w", "5w", "6w"],
      ["7t", "8t", "9t"],
      ["2s", "2s", "2s"],
      ["5s", "5s", "5s"]
    ],
    "pair": ["east", "east"]
  }
}
```

### 成功回應 (未胡牌)

*   **Status Code:** `200 OK`

```json
{
  "is_winning": false,
  "message": "尚未胡牌。",
  "reason": "缺乏對子或面子組合不正確。"
}
```

### 錯誤回應

*   **Status Code:** `422 Unprocessable Entity` (參數驗證失敗)

```json
{
  "error": "牌數不正確，台式麻將胡牌需為 17 張牌。"
}
```

## 5. 邏輯判斷標準 (台式麻將)

1.  **牌數檢查:** 手牌總數必須為 17 張（包含門清或碰、吃、槓後的組合）。
2.  **五面子一對子:**
    *   **面子 (Mianzi):** 可以是「順子」（同花色連續三張）或「刻子」（相同三張）。
    *   **對子 (Pair):** 兩張相同的牌。

## 6. 範例案例 (Example Cases)

### 成功案例 1：全順子 (All Chow)

*   **說明:** 手牌由五組順子與一組對子組成。
*   **請求:**
```json
{
  "hand": ["1w", "2w", "3w", "4w", "5w", "6w", "7w", "8w", "9w", "1t", "2t", "3t", "4t", "5t", "6t", "red", "red"]
}
```
*   **回應:**
```json
{
  "is_winning": true,
  "message": "恭喜胡牌！",
  "combination": {
    "mianzi": [
      ["1w", "2w", "3w"],
      ["4w", "5w", "6w"],
      ["7w", "8w", "9w"],
      ["1t", "2t", "3t"],
      ["4t", "5t", "6t"]
    ],
    "pair": ["red", "red"]
  }
}
```

### 成功案例 2：全刻子 (碰碰胡)

*   **說明:** 手牌由五組刻子與一組對子組成。
*   **請求:**
```json
{
  "hand": ["1s", "1s", "1s", "9s", "9s", "9s", "east", "east", "east", "west", "west", "west", "red", "red", "red", "white", "white"]
}
```
*   **回應:**
```json
{
  "is_winning": true,
  "message": "恭喜胡牌！",
  "combination": {
    "mianzi": [
      ["1s", "1s", "1s"],
      ["9s", "9s", "9s"],
      ["east", "east", "east"],
      ["west", "west", "west"],
      ["red", "red", "red"]
    ],
    "pair": ["white", "white"]
  }
}
```

### 失敗案例 1：牌數錯誤 (16張)

*   **說明:** 提供的牌數不足 17 張。
*   **請求:**
```json
{
  "hand": ["1w", "2w", "3w", "4w", "5w", "6w", "7t", "8t", "9t", "2s", "2s", "2s", "5s", "5s", "5s", "east"]
}
```
*   **回應 (422):**
```json
{
  "error": "牌數不正確，台式麻將胡牌需為 17 張牌。"
}
```

### 失敗案例 2：無法湊成對子

*   **說明:** 牌數雖然為 17 張，但無法組成「五面子 + 一對子」。
*   **請求:**
```json
{
  "hand": ["1w", "2w", "3w", "4w", "5w", "6w", "7w", "8w", "9w", "1t", "2t", "3t", "4t", "5t", "6t", "7t", "8t"]
}
```
*   **回應:**
```json
{
  "is_winning": false,
  "message": "尚未胡牌。",
  "reason": "缺乏對子或面子組合不正確。"
}
```
